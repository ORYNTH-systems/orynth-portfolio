# Reference Laboratories

## Overview

ORYNTH reference laboratories provide executable evidence for constitutional architectures.

Each lab translates a specification into deterministic cases, runtime behavior, benchmark corpora, or simulation demonstrations.

---

# Active Reference Labs

## CIGCS Reference Lab

Repository: https://github.com/ORYNTH-systems/cigcs-reference-lab

Version: v1.0.0

Status: Frozen benchmark

Purpose:

- Validate CIGCS constitutional behavior.
- Provide deterministic executable proof corpus.
- Establish benchmark evidence for admissibility, safety, Render Authorization Artifacts, and constitutional enforcement.

Corpus:

- Volume I: CB-001 through CB-100 — Constitutional Admissibility
- Volume II: SB-001 through SB-100 — Safety Predicate
- Volume III: RAA-001 through RAA-100 — Render Authorization Artifact
- Volume IV: CE-001 through CE-100 — Constitutional Enforcement

Total executable constitutional proofs: 400

## Simulation Integrity Lab

Repository: https://github.com/ORYNTH-systems/simulation-integrity-lab

Version: v0.1.0

Status: Active research runtime prototype

Purpose:

- Demonstrate constitutional governance over evolving simulation worlds.
- Evaluate admissibility during tick-based execution.
- Support replay, branching, drift analysis, multi-agent governance, distributed execution, counterfactual planning, and constitutional search.

Current capabilities:

- Tick-based simulation
- Runtime constitutional evaluation
- Scheduled world mutation
- Deterministic replay
- Branching timelines
- Constitutional drift analysis
- Multi-agent evaluation
- Negotiation and arbitration
- Distributed constitutional execution
- Counterfactual planning
- Constitutional search
